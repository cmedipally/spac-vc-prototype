The Problem

Admin controls live in the wrong place

Today
  • Multi-instance Viva Connections management lives in the legacy Viva blade in M365 Admin Center (MAC).
  • Admins expect to manage SharePoint properties in SharePoint Admin Center (SPAC) — that's where Home Sites already live for single-instance tenants.
  • The split creates friction and confusion, especially as single-instance and multi-instance flows diverge.
  • MAC also surfaces deprecated controls — Draft/Enabled status and License Type filtering — that are no longer meaningful.
The Ask
  • Move all multi-instance management to SPAC.
  • Deprecate legacy controls that add noise without value.
  • Establish SPAC as the single, authoritative place for Home Site administration.
Why Now
  • Lightweight VC instances are being retired — all experiences become full Home Sites.
  • This creates a natural forcing function to consolidate admin controls in SPAC.

The Plan

Three workstreams

✅ Enable in SPAC
  • Create multiple Home Sites from scratch or from an existing communication site
  • Audience targeting per instance (Azure AD security groups / M365 Groups)
  • Priority / fallback order across instances (Change Order)
  • Logo display control per instance
  • Port existing MAC configurations to SPAC — no admin action required
🔁 Remove from MAC
  • Remove multi-instance creation and management from the Viva blade in MAC once SPAC reaches parity
  • Replace with a redirect callout pointing admins to SPAC
  • Install and Pin in Teams remain in MAC (out of scope)
🗑 Deprecate
  • Lightweight VC instances — all upgraded to full Home Sites automatically, no admin action
  • Draft / Enabled status toggle — removed; all Home Sites are active by default
  • License Type filtering — not carried forward to SPAC

Risks & Open Decisions

What needs attention

🔴
R2a + R2b must ship together
Upgrading LW instances to full Home Sites automatically surfaces logos for end users. Without logo display control (R2b), admins cannot prevent unexpected visual changes. These two must be coordinated.
🔴
License Type filtering — no usage signal
No telemetry events are currently logged for License Type filtering (R1b). Engineering needs a deep dive to quantify usage and assess deprecation impact before we commit.
🟡
MAC removal timing
MAC multi-instance controls must not be removed until SPAC is at parity and admins have had sufficient notification time. Admin banner in MAC must go live by May 18 regardless of other timelines.
🟢
R2b (logo control) can be a fast follow
If R2a ships first, logo display behavior change may not be significant enough to block. Can be validated with early ring feedback and shipped as a fast follow if needed.
