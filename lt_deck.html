Leadership Review · May 2026
Migrating Viva Connections
Admin Controls to SharePoint Admin Center
Today, admins who manage multiple Viva Connections experiences are sent to a legacy blade in M365 Admin Center. We are proposing to consolidate this into SharePoint Admin Center — where they already manage everything else.
Viva Connections PM · cmedipally@microsoft.com
Background
What is this and why does it matter?
What is Viva Connections?
  • Viva Connections is Microsoft's employee experience app in Teams — giving employees a personalized entry point to their company intranet, news, and resources.
  • Admins configure it by designating a Home Site — a SharePoint communication site that powers the experience.
  • Larger organizations use multiple Home Sites to serve different employee audiences (e.g. global vs. frontline vs. regional).
The Problem Today
  • Single Home Site: managed in SharePoint Admin Center (SPAC).
  • Multiple Home Sites: managed in a legacy Viva blade in M365 Admin Center (MAC).
  • This split is confusing for admins and creates duplicated, inconsistent experiences.
What Are We Proposing?
  • Move all multi-instance Viva Connections management into SPAC — the single, authoritative place for SharePoint and Home Site administration.
  • Once parity is reached, remove the multi-instance management experience from MAC.
  • Alongside this migration, propose deprecating legacy controls that no longer serve a purpose.
💡
This is a migration, not a new feature
The goal is consolidation and simplification — bring existing functionality to where admins expect it, and clean up what no longer applies.

Proposal Overview
Three workstreams
Enable in SPAC
  • Create multiple Home Sites (new site or existing communication site)
  • Assign target audience per Home Site (Azure AD groups / M365 Groups)
  • Set priority and fallback order across instances
  • Control logo display per Home Site
  • Existing MAC configurations automatically visible in SPAC on day one — no admin re-entry
Remove from MAC
  • Once SPAC reaches feature parity, remove multi-instance management from the Viva blade in MAC
  • Replace with a redirect banner pointing admins to SPAC
  • Install and Pin in Teams remains in MAC (out of scope for this project)
Proposed Deprecations Proposed
  • Lightweight VC instances — propose upgrading all to full Home Sites automatically
  • Draft / Enabled status toggle — propose removing; all Home Sites active by default
  • License Type filtering — propose not carrying forward to SPAC

Proposed Deprecations Pending Alignment
What we propose to remove
⚠️
These are proposals, not decisions
The items below are proposed deprecations that require alignment. Each has a rationale and a known risk or dependency that needs resolution before we commit.
What Why deprecate Risk / Dependency Priority
Lightweight VC instances Lightweight instances are a legacy creation path. All new experiences will be full Home Sites. Existing LW instances would be auto-upgraded — no admin action needed. Auto-upgrade may surface logos in Viva Connections that admins didn't set up. Logo display control (P2) should ship alongside this to let admins manage the change. P0
Draft / Enabled status toggle This toggle no longer serves a purpose — visibility is now controlled by audience assignment, not status. Removing it reduces admin confusion. Admins must be notified before the toggle disappears. A MAC banner is planned for May 18. P0
License Type filtering Filtering by Everyone / Frontline / Information Worker is being replaced by group-based audience targeting, which is more flexible and already supported in SPAC. No telemetry events are logged for this feature today — we don't know how many admins actively use it. Engineering deep dive required before we can commit to deprecation. P0

Asks for Leadership
Where we need alignment
🔴
Align on License Type deprecation before committing
We have no telemetry on how many admins actively use License Type filtering. We need an engineering deep dive to understand impact before we formally deprecate. Should we proceed or hold?
🔴
Confirm LW deprecation and logo control ship together
Upgrading lightweight instances to full Home Sites will automatically surface logos in Viva Connections. Logo display control (P2) must ship alongside to avoid end-user surprise. Needs agreement to not slip one without the other.
🟡
Confirm MAC removal timing
Multi-instance controls in MAC should not be removed until SPAC is at full parity and admins have had adequate lead time. Admin banner in MAC must go live May 18 as the first signal to admins — this milestone is independent of everything else.
🟢
Logo display control (P2) can be a fast follow if needed
If LW deprecation ships first and end-user reaction to logo display is minimal, logo control can follow as a fast follow based on feedback. Agreement needed on this contingency.
